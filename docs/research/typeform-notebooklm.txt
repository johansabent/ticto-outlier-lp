### 1. Front-End: First-Touch Attribution & Typeform Embed

**Critique:** Relying on the standard iframe URL parameters is an architectural anti-pattern for single-page applications (SPAs). If a user browses the site before landing on the form page, the query parameters drop from the URL, leading to lost attribution data.

**Improvement (The "First-Touch" Strategy):**
To ensure absolute data continuity, we must capture the 7 required tracking parameters (`utm_source`, `utm_medium`, `utm_campaign`, `utm_content`, `utm_term`, `sck`, `src`) on the user's initial visit and cache them via `localStorage`. 

When instantiating the Typeform, you must dynamically inject these cached parameters into the form. Typeform provides two primary methods for this:
1.  **HTML/Vanilla JS:** Using the `data-tf-transitive-search-params` attribute forces the iframe to inherit URL query parameters. However, since we are fetching from `localStorage` rather than relying on the active URL, we must pass the parameters directly using the `hidden` object property.
2.  **React Library:** Use `@typeform/embed-react` to programmatically pass the hidden fields retrieved from local storage.

```tsx
// src/components/LeadForm.tsx
'use client';
import { Widget } from '@typeform/embed-react';
import { useEffect, useState } from 'react';

export function LeadForm({ formId }: { formId: string }) {
  const [hiddenFields, setHiddenFields] = useState<Record<string, string>>({});

  useEffect(() => {
    // Retrieve "first-touch" parameters cached during the initial page load
    setHiddenFields({
      utm_source: localStorage.getItem('utm_source') || 'organic',
      utm_campaign: localStorage.getItem('utm_campaign') || 'none',
      // ... map remaining parameters
    });
  }, []);

  return (
    <div className="w-full h-[600px]">
      <Widget 
        id={formId} 
        hidden={hiddenFields} 
        className="w-full h-full" 
      />
    </div>
  );
}
```

### 2. Back-End: Webhook Security & Payload Ingestion

**Critique:** Typeform webhooks send complex JSON payloads. Securing this endpoint is paramount. You cannot blindly parse the JSON payload without first verifying the cryptographic signature.

**Improvement (Strict HMAC-SHA256 Validation):**
Typeform sends an HMAC SHA-256 signature in the `Typeform-Signature` header, formatted as `sha256=<base64_encoded_HMAC>`. 
*Crucial Constraint:* The hash must be computed using your webhook secret against the **raw binary request body**, NOT the parsed JSON object. In Next.js 16 Route Handlers, you must extract the raw body using `await request.text()` before parsing.

```typescript
// src/app/api/webhooks/typeform/route.ts
import { NextResponse } from 'next/server';
import crypto from 'node:crypto';

export async function POST(request: Request) {
  try {
    // 1. Read raw body for HMAC validation
    const rawBody = await request.text();
    const signature = request.headers.get('Typeform-Signature');

    if (!signature) {
      return NextResponse.json({ error: 'Missing signature' }, { status: 401 });
    }

    // 2. Compute expected signature
    const secret = process.env.TYPEFORM_WEBHOOK_SECRET!;
    const expectedSig = 'sha256=' + crypto
      .createHmac('sha256', secret)
      .update(rawBody) // Raw payload is strictly required here
      .digest('base64');

    // 3. Constant-time comparison to prevent timing attacks
    if (!crypto.timingSafeEqual(Buffer.from(signature), Buffer.from(expectedSig))) {
      return NextResponse.json({ error: 'Invalid signature' }, { status: 401 });
    }

    // 4. Parse payload safely
    const payload = JSON.parse(rawBody);
```

### 3. CRM Integration: Idempotency & Defensive Mapping (Datacrazy)

**Critique:** You must execute the Datacrazy POST synchronously. Wrapping the main CRM execution in a `waitUntil` block is an anti-pattern that hides upstream failures and breaks Typeform's automated webhook retry mechanism.

**Improvement:**
1.  **Idempotency:** The Typeform payload contains a unique `event_id`. Since the Datacrazy API lacks native `X-Idempotency-Key` support, you must check this `event_id` against a fast key-value store (like Vercel KV) to ensure duplicate webhook deliveries are discarded.
2.  **Defensive Mapping:** The Datacrazy REST schema does not natively support custom fields for UTMs. You must extract the `hidden` object from `payload.form_response.hidden` and map these values defensively into Datacrazy's `tags` array (to retain CRM searchability) and dump the raw payload into the `notes` object for auditability.

```typescript
    // Continuing from the Route Handler above...
    
    const eventId = payload.event_id;
    // Check Vercel KV here to see if eventId was already processed to enforce idempotency

    const { hidden, answers } = payload.form_response;
    
    // Extract hidden fields from Typeform payload
    const source = hidden?.utm_source || 'organic';
    const campaign = hidden?.utm_campaign || 'none';

    // Extract basic fields (requires mapping answer IDs based on your Typeform definition)
    const emailAnswer = answers.find((a: any) => a.type === 'email')?.email;
    const nameAnswer = answers.find((a: any) => a.type === 'text')?.text;

    // Defensive Mapping for Datacrazy limitations
    const datacrazyPayload = {
      name: nameAnswer,
      email: emailAnswer,
      tags: [{ id: `source:${source}` }, { id: `campaign:${campaign}` }],
      notes: { raw_typeform_payload: JSON.stringify(payload) }
    };

    // Synchronous execution (Do NOT use waitUntil here)
    const crmResponse = await fetch('https://api.g1.datacrazy.io/api/v1/leads', {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
        'Authorization': `Bearer ${process.env.DATACRAZY_TOKEN}`,
      },
      body: JSON.stringify(datacrazyPayload),
    });

    if (!crmResponse.ok) {
      // Return 500 so Typeform knows the delivery failed and will queue a retry
      throw new Error('Datacrazy API rejected the lead');
    }

    return NextResponse.json({ success: true }, { status: 201 });
  } catch (error) {
    console.error('[Webhook Error]', error);
    return NextResponse.json({ error: 'Webhook processing failed' }, { status: 500 });
  }
}
```
